Meeting Cost Calculator - Development Prompts
==============================================

This document lists all the prompts used to create and refine the meeting cost calculator.

## Initial Creation

1. Create the basic calculator
   "lets create a new html page. The page will be a meeting cost calculator. it should have an input for a blended salary that defaults to $80 an hour. it should have an input for number of people on the call. it should have an input for time on call, HH:MM:SS. time on call should have a start and stop button that allows time to increment after or instead of the user providing initial input. From there the meeting cost should be calculated as the number of people times the blended salary times the elapsed time. account for the blended salary being per hour when multiplying in the elapsed time."

2. Documentation & linking
   "update documentation and link to the new page from the index"

15. Fix achievement timing

Summary
-------
These prompts show the iterative development process from initial creation through multiple rounds of refinement, bug fixes, and feature enhancements. The development progressed through:
- Basic functionality (inputs, timer, calculations)
- Visual enhancements (animations, responsive layout)
- Interactive features (achievements, presets)
- Polish and refinements (alignment, calibration, bug fixes)
